# Техническое задание: TastyAiOnlineTests

## 1. Общие сведения

- **Наименование:** Платформа для онлайн-тестирования с интеграцией ИИ (TastyAiOnlineTests)
- **Стек технологий:**
  - Бэкенд: Python 3.12+, Flask, Flask-Login, Flask-WTF
  - База данных: SQLite + SQLAlchemy (или raw sqlite3)
  - Фронтенд: HTML5, CSS3, JavaScript (Vanilla JS), Bootstrap 5 (или простой кастомный CSS)
  - ИИ-модуль: интеграция через API

## 2. Функциональные требования

### 2.1. Роли пользователей

- **Гость (неавторизованный):** может просматривать главную страницу, регистрироваться, входить
- **Пользователь (авторизованный):**
  - Создавать тесты
  - Проходить тесты (свои и чужие)
  - Просматривать свои тесты и результаты других пользователей по своим тестам
  - Управлять своими тестами (редактировать/удалять)
- **Администратор (опционально):** просмотр всех тестов и результатов

### 2.2. База данных (SQLite)

Таблицы: users, tests, attempts и т.д.

### 2.3. Модуль ИИ — два режима работы

#### Режим А: «Проверка правильных ответов»

- Пользователь отвечает на вопросы теста
- На сервер отправляются: user_answer + correct_answer (из БД) + question_text
- ИИ возвращает: правильно/неправильно + краткое пояснение (почему)
- Альтернатива без ИИ: сравнивается строго user_answer с correct_answer (регистр/пробелы нормализуются)

#### Режим Б: «Текстовый ответ на основе прохождения»

- После завершения теста пользователь может запросить персональный текстовый отзыв
- ИИ получает:
  - список вопросов пользователя
  - его ответы
  - правильные ответы (если есть)
  - итоговый процент
- ИИ возвращает связный текст (2–5 предложений) с рекомендациями и оценкой знаний

### 2.4. Страницы и их логика

1. **Главная (титульная страница)**
   - Краткое описание платформы
   - Кнопки: Вход / Регистрация (если не авторизован)
   - Если авторизован: приветствие, ссылки на: «Создать тест», «Мои тесты», «Доступные тесты»

2. **Регистрация / Авторизация**
   - Поля: логин, пароль
   - После входа — перенаправление на главную

3. **Создание теста**
   - Форма:
     - Название теста
     - Описание
     - Динамическое добавление вопросов (JavaScript):
       - Текст вопроса
       - Тип (закрытый / открытый)
       - Правильный ответ (для закрытого — текст ответа; для открытого — эталон)
   - Сохранение теста в БД (Flask, SQLite)

4. **Прохождение теста**
   - Отображаются все вопросы
   - После отправки:
     - Выбирается режим ИИ (или без ИИ):
       - Только проверка (сравнение с эталоном / ИИ)
       - Полный текстовый отзыв после завершения
   - Результаты сохраняются в attempts и user_answers
   - Вывод результата (баллы, %)
   - Если выбран текстовый отзыв — запрос к ИИ и показ ответа

5. **Мои тесты + результаты пользователей**
   - Список тестов, созданных текущим пользователем
   - Для каждого теста — кнопка «Результаты»
   - При переходе: таблица со всеми попытками других пользователей:
     - Кто проходил (логин)
     - Дата прохождения
     - Результат (%)
     - Режим ИИ
     - Детальный просмотр ответов пользователя

6. **Доступные тесты (все тесты всех авторов)**
   - Список тестов с кнопкой «Пройти»
   - Поиск тестов через текстовые запросы
   - Фильтрация

## 3. Требования к реализации

### 3.1. Бэкенд (Flask)

- **Маршруты (роутинг):**
  - `/` – главная
  - `/register`, `/login`, `/logout`
  - `/create-test` (GET, POST)
  - `/test/<id>` (GET — прохождение, POST — отправка ответов)
  - `/my-tests` – список тестов текущего пользователя
  - `/my-tests/<test_id>/results` – результаты по тесту
  - `/tests` – все доступные тесты
- **Хэндлер ИИ:** отдельный модуль `ai_service.py` с функциями:
  - `check_answer_with_ai(question, user_answer, correct_answer) -> (is_correct, comment)`
  - `generate_text_feedback(questions, user_answers, correct_answers, score_percent) -> str`
- При недоступности ИИ — падение на режим «без ИИ» (сравнение строк)

### 3.2. Фронтенд (JavaScript)

- Динамическое добавление полей вопросов на странице создания теста

- Отправка форм через `fetch` (AJAX) или обычный POST с перезагрузкой
- Отображение спиннера при ожидании ответа от ИИ

### 3.3. Безопасность и куки

- Хранение паролей: хеширование
- Сессии: Flask-Login с куками
- Защита от CSRF: Flask-WTF (для форм создания теста)
- Пользователь не должен видеть чужие результаты тестов (кроме автора теста)
